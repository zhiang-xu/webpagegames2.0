# 张七七的游戏乐园 — Agent 工作规范

本文档为 AI Agent（Claude）在此项目中的工作规范和约定。所有 Agent 操作必须遵循本文件。

---

## 1. 代码修改流程

每次收到代码修改需求，必须遵循以下流程：

1. **理解需求** — 充分理解用户想要实现的目标
2. **制定计划** — 列出具体的修改步骤，与用户确认后再执行
3. **实施修改** — 按计划逐步修改代码
4. **更新文档** — 同步更新相关 README 文件（详见第 4 节）
5. **测试验证** — 在桌面端和移动端分别测试核心功能
6. **提交推送** — 按规范提交 Git 并推送

---

## 2. 版本号规范

遵循 **语义化版本（SemVer）**：`主版本.次版本.修订号`

| 级别 | 适用场景 | 示例 |
|------|----------|------|
| **主版本**（破坏性改动） | 重写核心逻辑、改变游戏规则、变更技术架构 | v2.0.0 |
| **次版本**（新功能/重大改进） | 新增游戏、新增功能模块、显著提升 AI 强度/体验 | v1.1.0 |
| **修订号**（日常维护） | Bug 修复、UI/UX 微调、样式优化、文档更新、代码重构（不影响功能） | v1.0.1 |

- 所有游戏初始版本统一为 **v1.0.0**（表示初次发布）
- 每次发布后版本号只增不减

## 3. 代码风格规范

- **HTML**：语义化标签，`lang="zh-CN"`，UTF-8 编码
- **CSS**：
  - 类名使用 `kebab-case`（如 `game-board`）
  - 颜色、间距等常用值尽量抽取为 CSS 变量或统一定义
  - 响应式断点以 700px 为移动端分界线
- **JavaScript**：
  - 函数名、变量名使用 `camelCase`
  - 游戏逻辑使用 IIFE 或函数封装，避免污染全局命名空间
  - 不滥用全局变量
- **文件结构**：各游戏独立目录（`games/游戏名/`），内含 `index.html`、`index.css`、`index.js`；共享代码放 `js/` 目录，共享样式放 `css/` 目录

---

## 4. 文档更新规范（核心）

> **每次代码修改后，必须同步更新对应文档。**

### 更新规则

| 修改类型 | 根目录 `readme.md` | 游戏 `readme_*.md` |
|----------|-------------------|-------------------|
| 新增游戏 | 更新「游戏列表」表格 | 新建对应 `readme_*.md` |
| 新增功能/改动 | 在版本历史中记录 | 在版本历史中记录 |
| Bug 修复 | 可选记录 | 记录修订号变更 |
| 样式/UI 优化 | 可选记录 | 记录修订号变更 |
| 重构（不影响功能） | 可选记录 | 记录修订号变更 |
| 文档更新 | 必须更新版本历史 | 必须更新版本历史 |

### 更新时机

- **新增游戏**：创建 `游戏文件夹/readme_*.md`，在根目录 `readme.md` 的「游戏列表」中添加条目
- **新增功能**：在对应 `readme_*.md` 的 `## 版本历史` 中新增条目，版本号按修改级别递增
- **修改共享组件**（`js/`、`css/`）：所有受影响游戏的 `games/游戏名/readme_*.md` 均需记录修订号变更

### 版本历史格式

```markdown
### v1.2.0 (2026-06-21)
- 新增功能：xxx
- 优化：xxx
- 修复：xxx
```

## 5. 音效系统规范

- 所有游戏统一使用 `js/game-ui.js` 提供的 `AudioManager`
- **不**在游戏内单独硬编码音效
- 新增音效类型需同步更新 `js/game-ui.js` 的 `patterns` 对象，并在本文档第 5 节（音效类型列表）中登记
- 当前音效类型：`tap`、`select`、`move`、`drop`、`clear`、`success`、`win`、`error`、`lose`、`pause`、`start`

---

## 6. UI 组件规范

- 所有游戏页面必须集成 `js/game-ui.js`，提供统一的：
  - 浮动返回主页按钮（左侧）
  - 浮动音效开关按钮（右侧）
- 移动端兼容性要求：
  - 触屏滑动支持（使用 `touch` 事件或 Pointer Events API）
  - 按钮尺寸不小于 44px（触摸友好）
- 响应式布局：主要断点为 700px

---

## 7. 性能规范

- Canvas 游戏循环使用 `requestAnimationFrame`，避免 `setInterval`
- 页面切换时清理不再需要的事件监听器，防止内存泄漏
- 图片/字体等静态资源优先使用 CDN 或公共资源，避免仓库体积过大

---

## 8. Git 提交规范

- 提交信息使用**中文**，格式：`类型: 简短描述`
- 类型前缀：`feat`（功能）、`fix`（修复）、`refactor`（重构）、`docs`（文档）、`style`（样式）、`perf`（性能）
- 每次提交尽量保持**原子性**（一个提交只做一件事）
- 示例：
  ```
  feat: 新增数独游戏提示功能
  fix: 修复贪吃蛇撞墙后音效未停止的问题
  docs: 更新五子棋版本历史
  refactor: 统一俄罗斯方块按键响应逻辑
  ```

---

## 9. 测试规范

每次新增游戏或重大功能发布前，必须在以下环境测试：

| 环境 | 测试内容 |
|------|----------|
| 桌面端（Chrome/Edge） | 键盘操作、鼠标点击、音效播放 |
| 移动端（手机浏览器） | 触屏滑动、按钮点击、音效播放、布局适配 |
| 触屏设备（平板） | 布局是否合理，按钮是否足够大 |

测试重点：核心游戏流程、音效、触屏交互、localStorage 读写

---

## 10. 新游戏接入规范

新增游戏时需要创建/修改以下文件：

| 文件 | 操作 |
|------|------|
| `games/游戏名/index.html` | 创建游戏主文件 |
| `games/游戏名/readme_*.md` | 创建文档，版本 v1.0.0 |
| `index.html`（根目录） | 在「游戏列表」中添加条目，添加链接 |
| `readme.md`（根目录） | 在「游戏列表」中添加条目 |
| `js/game-ui.js` | 如需新增音效类型，在此更新 |
